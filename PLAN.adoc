* [ ] P1.3 `tools`: thin tool-dispatch registry (`ToolCall` -> result), one trivial built-in tool, unit-tested

* [ ] P1.6 `pump`: manual agent loop -- call model, dispatch tools, feed results, repeat; paces turns, carries no intelligence
* [ ] P1.7 Wire pump + tui + model + tools end to end; `coxn` runs the loop against the stub backend (MVP exit criterion)
